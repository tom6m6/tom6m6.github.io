    信息存储

    对于计算机来说，最最小的单位是一个又一个的位（高电压/低电压，只有两种可能0/1），但是这样的话读取和使用起来并不是十分便捷，所以将8个位（bit）组成了一个块，即字节（byte）。作为最小的可寻址的内存单位，机器级程序 将很多的字节（bytes）视为一个非常大的数组（虚拟内存 virtual memory），这个数组的每一个元素都有唯一对应的数字（即，地址）,可能的地址们又组成了虚拟地址空间。

    C语言中的指针有两个方面，值（存的地址）和类型（指向的东西的类型）

    十六进制表示法

    一个字节有8位，如果用二进制来表示，特别偏重于底层，但是也特别冗长。如果用十进制来表示，表达地简略了一些，但是在涉及到位运算时十分麻烦。采取一个折衷的办法，采用十六进制作为表示数的主流方式。
    十六进制，简写为hex.在 C 语言中，以 0x 或 0X 开头的数字常量被认为是十六进制的值。

    一个十六进制位有16种可能（0-F）但一个二进制位只有两种可能，只有四个二进制位才有16种可能，因此一个十六进制位可以表示四个二进制位。一个字节最多包含两个十六进制位。

    二进制和十六进制的转换十分简单，只需要知道对应关系，其实就是第几个对应另外的那个的第几个，比如0x1对应0001，0xA对应1011。如果二进制缺位，则自动补0，这样就可以实现两者的相互转换。

    二进制对十进制的转换有那么一些复杂。
    对于十进制整数来说，一直除2并取模，直到1停止，取得的模数最后反过来便是其二进制。比如十进制整数6，按上述操作分别得到的模数是0，1，1，然后反过来（110）便是其二进制。
    对于十进制小数来说，其整数部分按照上述的操作。但是对于小数部分，每次取出小数点后的部分，然后乘2，而整数部分留了下来，最后依次（不是反过来）得到的整数便是二进制。比如0.8125

    字数据大小

    每台计算机有一个字长，表明指针数据的标称大小，而虚拟地址也是以这样的字来编码的，字长也影响了虚拟地址的最大大小，这是十分重要的。
    对于k位的机器，最多能指向2^k种可能情况，故虚拟地址的范围是0~2^k-1，程序最多访问2^k个字节。
    在表示的大小确定的情况下，这里要考虑有符号和无符号的问题，因为会影响取值。
    以下这四句话一个意思，C语言对这种的前后顺序不敏感。
    unsigned long
    unsigned long int
    long unsigned
    long unsigned int
    int现在都认为占4个字节
    long在32位机器占4字节，在64位机器上占8字节。
    char 在32机器上占4字节，在64位机器上占8字节。
    （讲真，这个char     的大小正好和两种机器的空间对应了起来，真的奇妙。）
    float在32位机器占4字节，在64位机器上占4字节。
    double在32位机器占8字节，在64位机器上占8字节。

    寻址和字节顺序

    我们想存很多很多的东西，但是一个字节只有256种可能，这显然是不够的，我们只好跨越多个字节来存。
    在几乎所有的机器上，多字节对象都被存储为连续的字节序列，对象的地址为所使用字节中最小的地址。
    根据上面讲到的，一个字节能存两个二进制位。假设有一个int类型的数0x12345678，我们怎样存储呢？
    首先明确，12这一端是最高有效字节，另外一端是最低有效字节。
    大端法，是先高后低，存储之后是这样的 12、34、56、78
    小端法，反了过来，是这样的，78、56、34、12
    两种方法各有千秋

    逻辑运算和位运算

    按位与& 两个都是1才为1
    按位或| 两个都为0才为0
    按位取反~ 1变成0 0变成1
    按位异或^ 两个不一样才为1，一样就为0
    这个要和逻辑运算区分清楚了，逻辑运算是&&，||这样的，有相似之处，但要明白差异
    左移k<<4 右移k>>4
    左移很简单，是丢掉左边超出的，然后右边空余的地方补0
    右移有两种，逻辑右移（左端补0）和算术右移（左端补k个最高有效位的值）
    一般有符号数使用算术右移，但是无符号数使用逻辑右移。
    有一个极端情况，如果移动的位数k（k>=w）特别特别大怎么办？在这样的情况下，实际只会移动 k mod w位.
    注意C语言种移位运算符的优先级不如+/-运算符，使用时要带括号。

    整数表示

    一个很值得注意的特点是取值范围不是对称的负数的范围比整数的范围大 1。比如char的取值范围是-128~127
    无符号数的编码最简单了，只用二进制和十进制之间的基本转换就可。
    最常见的负数表达形式是补码形式。在此之中，将字的最高有效位称为负权。
    补码和无符号编码的比较如下。

    B2U是二进制变无符号
    B2T是二进制变有符号
    Umax最大无符号数 Tmax最大补码值

    补码的范围是不对称的，再次强调！
    反码的最高有效位的权是(-2^(w-1)-1)而不是(-2^(w-1))，其他的都一样，
    还是拿最高四位的-5来说话，-5的补码是1011，反码是1010

    原码就是最高有效位是符号位，0为无符号，1为有符号，其他仿照无符号数的准则。

    输出结果是-12345和53191，为什么？
    short占2个字节，16位，最高16位
    -12345的16位补码和53191的16位无符号表示是一样的。 -12345和53191之间正好差了两个2^15，一个是加的，一个是减的，当然是差了两个啊。

    为什么会出现这样的结果？
    首先，unsigned 可以代表unsigned int ,这样就是4字节，32位了，然后结合上述的方法，两者的码是一样的，然后就可以得到这样的结果。
    T2U代表的是将一个有符号的数转化为一个无符号的数
    此处的w指的是位数
    若x>=0,T2U(x)=x
    若x<0,T2U(x)=x+2*2^(w-1)=x+2^w
    U2T代表的是将一个无符号的数转化为一个有符号的数
    若0<=x<=Tmax,U2T(x)=x
    若x>Tmax,U2T(x)=x-2^w
